# Второй пак задач

### 1 задача
Реализовать функцию с переменным числом аргументов, которая для набора десятичных дробей из интервала (0;1) определяет, имеет ли каждая дробь конечное или бесконечное представление в системе счисления с заданным основанием. Необходимо использовать механизм функций с переменным числом аргументов и продемонстрировать работу программы.
### Решение
В программе реализована функция с переменным числом аргументов, принимающая основание системы счисления и набор дробных чисел. Основание анализируется на предмет разложения на простые множители 2 и 5. Для каждого числа выделяется дробная часть, удаляются незначащие нули, после чего проверяется возможность полного сокращения знаменателя на множители, отсутствующие в основании системы счисления. На основании результатов определяется, будет ли представление числа в заданной системе счисления конечным или бесконечным. Для работы с большими значениями используются строковые операции и алгоритмы деления чисел, представленных строками.
### 2 задача 
Реализовать собственные версии функций стандартной библиотеки языка C для работы с памятью и строками: `memchr`, `memcmp`, `memcpy`, `memset`, `strncat`, `strchr`, `strncmp`, `strncpy`, `strcspn`, `strerror`, `strlen`, `strpbrk`, `strrchr`, `strstr` и `strtok`. Также необходимо реализовать поддержку сообщений об ошибках для операционных систем Linux и macOS.

### Решение
Все требуемые функции реализованы без использования соответствующих функций стандартной библиотеки. Для функций работы с памятью применяются последовательный просмотр и копирование байтов. Строковые функции реализованы посредством посимвольного обхода строк, сравнения символов и поиска подстрок. Функция `strerror` возвращает текстовые описания ошибок из заранее подготовленных массивов сообщений, выбор которых выполняется с помощью директив условной компиляции в зависимости от используемой операционной системы. Реализация полностью повторяет базовую функциональность стандартной библиотеки и позволяет проверить принципы работы строк и областей памяти на низком уровне.

### 3 задача
Реализовать функции `overfprintf` и `oversprintf`, аналогичные стандартным функциям `fprintf` и `sprintf`, но с поддержкой дополнительных пользовательских спецификаторов формата. Дополнительные флаги должны обеспечивать вывод чисел в римской системе счисления, представлении Цекендорфа, произвольных системах счисления, преобразование строковых представлений чисел в десятичную систему счисления, а также вывод дампа памяти различных типов данных в двоичном виде. Необходимо продемонстрировать работу реализованных функций.

### Решение
В программе реализованы функции `overfprintf`, `oversprintf` и их внутренние версии, выполняющие разбор форматной строки и обработку как стандартных, так и пользовательских спецификаторов. Для флага `%Ro` реализовано преобразование целого числа в римскую запись с использованием таблицы соответствий арабских и римских чисел. Флаг `%Zr` формирует представление Цекендорфа на основе чисел Фибоначчи. Флаги `%Cv` и `%CV` выполняют перевод целых чисел в систему счисления с основанием от 2 до 36 с выводом букв соответственно в нижнем или верхнем регистре. Флаги `%to` и `%TO` осуществляют обратное преобразование строкового представления числа из произвольной системы счисления в десятичную. Для флагов `%mi`, `%mu`, `%mf` и `%md` реализован побайтовый дамп памяти соответствующих типов данных с выводом каждого байта в двоичном виде. Работа функций организована через механизм переменного числа аргументов (`stdarg.h`), что позволяет сохранить интерфейс и принципы использования стандартных функций форматированного вывода.
### 4 задача
Реализовать функции `overfscanf` и `oversscanf`, аналогичные стандартным функциям `fscanf` и `sscanf`, но с поддержкой дополнительных пользовательских спецификаторов формата. Дополнительные флаги должны обеспечивать считывание целых чисел, записанных римскими цифрами, в представлении Цекендорфа, а также в системах счисления с произвольным основанием от 2 до 36. Необходимо продемонстрировать работу реализованных функций.

### Решение
В программе реализованы функции `overfscanf`, `oversscanf` и их внутренние версии, выполняющие разбор форматной строки и обработку как стандартных, так и пользовательских спецификаторов. Для флага `%Ro` реализовано считывание числа, записанного римскими цифрами, с последующим преобразованием его в значение типа `int` на основе таблицы соответствий римских символов и их числовых значений. Флаг `%Zr` обеспечивает считывание числа в представлении Цекендорфа и вычисление его значения с использованием последовательности чисел Фибоначчи. Флаги `%Cv` и `%CV` выполняют считывание чисел, записанных в системе счисления с основанием от 2 до 36, и переводят их в десятичное представление; при этом `%Cv` обрабатывает буквенные символы в нижнем регистре, а `%CV` — в верхнем. Для реализации использован механизм переменного числа аргументов (`stdarg.h`), что позволяет сохранить интерфейс и принципы работы стандартных функций семейства `scanf`. Дополнительно реализованы вспомогательные функции для разбора римских чисел, представления Цекендорфа и чисел в произвольных системах счисления.

### 5 задача
Реализовать программу, которая в интерактивном режиме принимает пути к входному и выходному файлам, переносит текст из входного файла в выходной и выполняет форматирование строк. Каждая строка исходного текста должна разбиваться на строки длиной не более 80 символов таким образом, чтобы 80-й символ являлся последним символом некоторого слова. Для достижения необходимой длины между словами равномерно распределяются дополнительные пробелы. Если длина фрагмента меньше 80 символов, он переносится в выходной файл без изменений. Каждая новая строка должна начинаться с читаемого символа.

### Решение
В программе реализован набор функций для обработки и форматирования текстовых данных. С помощью функции `process_files` осуществляется открытие входного и выходного файлов, построчное чтение исходного текста и запись результата в новый файл. Для анализа содержимого строки используются функции `count_words` и `split_into_words`, которые определяют количество слов и разбивают строку на отдельные лексемы. Основная логика форматирования реализована в функции `format_line`, которая формирует строки длиной до 80 символов, подбирая максимально возможное количество слов и равномерно распределяя между ними дополнительные пробелы. Если длина исходной строки не превышает установленное ограничение, она копируется без изменений. Для определения границ слов используется функция `is_readable_char`, проверяющая принадлежность символа к печатным и отличным от пробела символам. В результате программа обеспечивает корректное выравнивание текста по ширине строки и сохранение читаемости исходного содержимого при переносе данных между файлами.
