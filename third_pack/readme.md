# Третий пак

### Задача 1
Реализовать функцию перевода числа из десятичной системы счисления в систему счисления с основанием (2^r), где (r = 1, \dots, 5) (основания 2, 4, 8, 16 и 32). При реализации разрешается использовать только битовые операции и операции обращения к памяти, использование стандартных арифметических операций запрещено. Необходимо продемонстрировать работу реализованной функции.

### Решения
В программе реализован набор функций для преобразования целого числа из десятичной системы счисления в системы счисления с основаниями 2, 4, 8, 16 и 32. Основная функция `convert_to_p` определяет требуемое основание и выполняет преобразование, используя двоичное представление числа. Для основания 2 используется отдельная функция `convert_to_2`, формирующая строку из битов числа посредством побитовых сдвигов и масок. Для остальных оснований число предварительно переводится в двоичный вид функцией `int_to_binary`, после чего биты группируются по (r) штук и преобразуются в соответствующие символы результирующей системы счисления. Все арифметические действия, необходимые для работы алгоритма, реализованы через функции `bit_add` и `bit_sub`, использующие исключительно побитовые операции. После формирования результата функция `remove_leading_zeros` удаляет незначащие ведущие нули. Дополнительно реализована функция `string_to_int`, выполняющая преобразование входной строки в целое число с проверкой корректности ввода и возможного переполнения. Такой подход полностью удовлетворяет ограничению на использование только битовых операций при выполнении преобразований.



### Задача 2
Реализовать библиотеку, предоставляющую функционал динамического массива (`Vector`) для произвольного типа данных с использованием макросов. В рамках одной программы допускается использование только одного типа элементов массива. При создании контейнера пользователь должен передавать функции копирования и удаления элементов, которые сохраняются внутри структуры. Требуется реализовать структуру `Vector` и набор функций для создания, копирования, сравнения, изменения и удаления динамического массива.

### Решение
В программе реализована структура `Vector`, содержащая указатель на динамический массив элементов, текущий размер, ёмкость массива, а также указатели на пользовательские функции копирования и удаления элементов. Создание контейнера выполняется функцией `create_vector`, которая инициализирует структуру и выделяет память под элементы. Для автоматического увеличения объёма памяти реализована вспомогательная функция `ensure_capacity`, удваивающая вместимость массива при необходимости. Добавление новых элементов осуществляется функцией `push_back_vector`, использующей переданную пользователем функцию копирования либо поверхностное копирование по умолчанию. Удаление элемента по индексу выполняется функцией `delete_at_vector` с последующим сдвигом оставшихся элементов. Функция `get_at_vector` обеспечивает доступ к элементу по индексу. Для сравнения двух контейнеров реализована функция `is_equal_vector`, выполняющая поэлементное сравнение содержимого. Копирование массива поддерживается как в уже существующий объект (`copy_vector`), так и в новый экземпляр, размещённый в динамической памяти (`copy_vector_new`). Очистка внутреннего содержимого и освобождение выделенной памяти выполняются функциями `erase_vector` и `delete_vector`, которые также вызывают пользовательскую функцию удаления элементов при её наличии. Использование макроса `VECTOR_TYPE` позволяет адаптировать библиотеку для работы с любым типом данных без изменения основной логики реализации.

### Задача 4
Реализовать библиотеку, предоставляющую функционал бинарной кучи (Heap). Требовалось определить структуру `Heap` и реализовать операции создания и удаления кучи, проверки на пустоту, получения размера, доступа к минимальному элементу без удаления, добавления и извлечения элементов, построения кучи из массива, а также сравнения двух куч по их внутреннему представлению.

### Решение
В работе реализована структура `Heap`, содержащая динамический массив элементов, текущий размер и вместимость. Функция `create_heap` создаёт пустую кучу заданной ёмкости, а `delete_heap` освобождает выделенную память. Добавление элементов выполняется функцией `push_heap`, которая при необходимости увеличивает размер массива и восстанавливает свойства минимальной кучи с помощью процедуры `heapify_up`. Извлечение минимального элемента реализовано функцией `pop_heap`, использующей процедуру `heapify_down`. Для быстрого получения минимального элемента без удаления предусмотрена функция `peek_heap`. Функция `build_heap` строит корректную кучу из массива элементов, а `is_equal_heap` выполняет лексикографическое сравнение внутреннего представления двух куч. Для управления памятью реализовано автоматическое расширение массива при нехватке места.

### Задача 6

Написать программу на языке C для проверки корректности расстановки скобок в строке. Входная строка может содержать произвольные символы, однако анализируются только круглые `()`, квадратные `[]`, фигурные `{}` и угловые `<>` скобки. Необходимо определить, является ли последовательность скобок сбалансированной, то есть каждой открывающей скобке соответствует закрывающая того же типа, а порядок их вложенности не нарушен.
### Решение
В программе реализована функция `check_brackets`, использующая стек для хранения открывающих скобок. При встрече открывающей скобки она помещается в стек, а при встрече закрывающей выполняется проверка соответствия последней открытой скобке. Если типы скобок не совпадают или стек оказывается пустым, строка считается некорректной. Для хранения элементов используется динамически расширяемый массив, что позволяет обрабатывать строки произвольной длины. После обработки всей строки функция проверяет, пуст ли стек: если да, то все скобки расставлены правильно, иначе в строке остались незакрытые скобки. В результате функция возвращает логическое значение, отражающее корректность скобочной структуры строки.

### Задача 7

Разработать консольное приложение на языке C, реализующее простой интерпретатор математических выражений. Интерпретатор должен поддерживать переменные с именами от `A` до `Z`, операции присваивания, арифметические операции `+`, `-`, `*`, `/`, быстрое возведение в степень `^`, а также функцию вывода `print(X)`. Каждая команда записывается с новой строки и может содержать пробелы. После обработки каждой команды необходимо сформировать запись в файл трассировки, содержащую исходную команду, значения всех инициализированных переменных и описание выполненной операции.

### Решение 
В программе реализован интерпретатор, выполняющий построчный разбор и обработку команд. Поддерживаются переменные с именами от `A` до `Z`, операции присваивания, арифметические действия `+`, `-`, `*`, `/`, операция возведения в степень `^` и функция `print` для вывода значений переменных. Значения переменных хранятся в памяти и обновляются по мере выполнения команд. После обработки каждой строки в файл трассировки записываются исходная команда, значения всех инициализированных переменных и описание выполненной операции, что позволяет контролировать ход выполнения программы и изменения состояния интерпретатора.
